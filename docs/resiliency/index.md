* <span data-ttu-id="af3fc-477">Ведение журнала приложения — это не то же, что и аудит.</span><span class="sxs-lookup"><span data-stu-id="af3fc-477">Application logging is not the same as auditing.</span></span> <span data-ttu-id="af3fc-478">Аудит может выполняться для соответствия требованиям или нормам.</span><span class="sxs-lookup"><span data-stu-id="af3fc-478">Auditing may be done for compliance or regulatory reasons.</span></span> <span data-ttu-id="af3fc-479">Таким образом, следует выполнить записи аудита, не прерывая запись при обработке транзакций.</span><span class="sxs-lookup"><span data-stu-id="af3fc-479">As such, audit records must be complete, and it's not acceptible to drop any while processing transactions.</span></span> <span data-ttu-id="af3fc-480">Если приложению требуется аудит, он должен выполняться отдельно от ведения журнала диагностики.</span><span class="sxs-lookup"><span data-stu-id="af3fc-480">If an application requires auditing, this should be kept separate from diagnostics logging.</span></span>

## <a name="manual-failure-responses"></a><span data-ttu-id="af3fc-482">Обработка сбоев в ручном режиме</span><span class="sxs-lookup"><span data-stu-id="af3fc-482">Manual failure responses</span></span>
<span data-ttu-id="af3fc-483">Предыдущие разделы посвящены автоматическим стратегиям восстановления, которые имеют решающее значение для обеспечения высокой доступности.</span><span class="sxs-lookup"><span data-stu-id="af3fc-483">Previous sections have focused on automated recovery strategies, which are critical for high availability.</span></span> <span data-ttu-id="af3fc-484">Однако иногда требуется ручное вмешательство.</span><span class="sxs-lookup"><span data-stu-id="af3fc-484">However, sometimes manual intervention is needed.</span></span>

* <span data-ttu-id="af3fc-485">**Оповещения.**</span><span class="sxs-lookup"><span data-stu-id="af3fc-485">**Alerts**.</span></span> <span data-ttu-id="af3fc-486">Отслеживайте приложение на предмет предупреждающих знаков, которые могут потребовать активного вмешательства.</span><span class="sxs-lookup"><span data-stu-id="af3fc-486">Monitor your application for warning signs that may require proactive intervention.</span></span> <span data-ttu-id="af3fc-487">Например, если вы видите, что база данных SQL или база данных Cosmos DB постоянно регулирует ваше приложение, вам может потребоваться увеличить емкость базы данных или оптимизировать ваши запросы.</span><span class="sxs-lookup"><span data-stu-id="af3fc-487">For example, if you see that SQL Database or Cosmos DB consistently throttles your application, you might need to increase your database capacity or optimize your queries.</span></span> <span data-ttu-id="af3fc-488">В этом примере, несмотря на то что приложение может обрабатывать ошибки регулирования прозрачно, ваша телеметрия должна все равно вызывать оповещение, чтобы вы могли предпринять ответные меры.</span><span class="sxs-lookup"><span data-stu-id="af3fc-488">In this example, even though the application might handle the throttling errors transparently, your telemetry should still raise an alert so that you can follow up.</span></span>  
* <span data-ttu-id="af3fc-489">**Отработка отказа вручную**.</span><span class="sxs-lookup"><span data-stu-id="af3fc-489">**Manual failover**.</span></span> <span data-ttu-id="af3fc-490">Некоторые системы не могут автоматически выполнять отработку отказа и требуют выполнения перехода на другой ресурс вручную.</span><span class="sxs-lookup"><span data-stu-id="af3fc-490">Some systems cannot fail over automatically and require a manual failover.</span></span> 
* <span data-ttu-id="af3fc-491">**Проверка готовности к работе**.</span><span class="sxs-lookup"><span data-stu-id="af3fc-491">**Operational readiness testing**.</span></span> <span data-ttu-id="af3fc-492">Если ваше приложение выполняет отработку отказа в дополнительный регион, следует запустить тест готовности к работе, прежде чем вы восстановите размещение в основной регион.</span><span class="sxs-lookup"><span data-stu-id="af3fc-492">If your application fails over to a secondary region, you should perform an operational readiness test before you fail back to the primary region.</span></span> <span data-ttu-id="af3fc-493">Тестирование должно проверить, что основной регион работоспособен и готов снова получать трафик.</span><span class="sxs-lookup"><span data-stu-id="af3fc-493">The test should verify that the primary region is healthy and ready to receive traffic again.</span></span>
* <span data-ttu-id="af3fc-494">**Проверка согласованности данных**.</span><span class="sxs-lookup"><span data-stu-id="af3fc-494">**Data consistency check**.</span></span> <span data-ttu-id="af3fc-495">Если сбой происходит в хранилище данных, может возникнуть несогласованность данных, когда хранилище снова станет доступным, особенно если данные были реплицированы.</span><span class="sxs-lookup"><span data-stu-id="af3fc-495">If a failure happens in a data store, there may be data inconsistencies when the store becomes available again, especially if the data was replicated.</span></span> 
* <span data-ttu-id="af3fc-496">**Восстановление из резервной копии**.</span><span class="sxs-lookup"><span data-stu-id="af3fc-496">**Restoring from backup**.</span></span> <span data-ttu-id="af3fc-497">Например, если в базе данных SQL происходит региональный сбой, вы можете выполнить геовосстановление базы данных из последней резервной копии.</span><span class="sxs-lookup"><span data-stu-id="af3fc-497">For example, if SQL Database experiences a regional outage, you can geo-restore the database from the latest backup.</span></span>

<span data-ttu-id="af3fc-498">Документируйте и протестируйте план аварийного восстановления.</span><span class="sxs-lookup"><span data-stu-id="af3fc-498">Document and test your disaster recovery plan.</span></span> <span data-ttu-id="af3fc-499">Оцените влияние сбоев приложения на бизнес.</span><span class="sxs-lookup"><span data-stu-id="af3fc-499">Evaluate the business impact of application failures.</span></span> <span data-ttu-id="af3fc-500">Максимально автоматизируйте процесс и документируйте любые шаги, выполняемые вручную, такие как переход на другой ресурс вручную или восстановление данных из резервных копий.</span><span class="sxs-lookup"><span data-stu-id="af3fc-500">Automate the process as much as possible, and document any manual steps, such as manual failover or data restoration from backups.</span></span> <span data-ttu-id="af3fc-501">Регулярно тестируйте процесс аварийного восстановления для проверки и улучшения плана.</span><span class="sxs-lookup"><span data-stu-id="af3fc-501">Regularly test your disaster recovery process to validate and improve the plan.</span></span>
