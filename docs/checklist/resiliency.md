<span data-ttu-id="eb1de-491">**Не выполняйте рабочую нагрузку на одной виртуальной машине.**</span><span class="sxs-lookup"><span data-stu-id="eb1de-491">**Avoid running a production workload on a single VM.**</span></span> <span data-ttu-id="eb1de-492">Одно развертывание виртуальной машины неустойчиво при плановом или внеплановом обслуживании.</span><span class="sxs-lookup"><span data-stu-id="eb1de-492">A single VM deployment is not resilient to planned or unplanned maintenance.</span></span> <span data-ttu-id="eb1de-493">Вместо этого поместите несколько виртуальных машин в группу доступности или [масштабируемый набор виртуальных машин](/azure/virtual-machine-scale-sets/virtual-machine-scale-sets-overview/) перед подсистемой балансировки нагрузки.</span><span class="sxs-lookup"><span data-stu-id="eb1de-493">Instead, put multiple VMs in an availability set or [VM scale set](/azure/virtual-machine-scale-sets/virtual-machine-scale-sets-overview/), with a load balancer in front.</span></span>

<span data-ttu-id="eb1de-494">**Укажите группу доступности при подготовке виртуальной машины.**</span><span class="sxs-lookup"><span data-stu-id="eb1de-494">**Specify an availability set when you provision the VM.**</span></span> <span data-ttu-id="eb1de-495">В настоящее время после подготовки виртуальную машину невозможно добавить в группу доступности.</span><span class="sxs-lookup"><span data-stu-id="eb1de-495">Currently, there is no way to add a VM to an availability set after the VM is provisioned.</span></span> <span data-ttu-id="eb1de-496">Добавляя новую виртуальную машину в имеющуюся группу доступности, обязательно создайте сетевую карту для виртуальной машины и добавьте ее в серверный пул адресов в подсистеме балансировки нагрузки.</span><span class="sxs-lookup"><span data-stu-id="eb1de-496">When you add a new VM to an existing availability set, make sure to create a NIC for the VM, and add the NIC to the back-end address pool on the load balancer.</span></span> <span data-ttu-id="eb1de-497">В противном случае подсистема балансировки нагрузки не будет маршрутизировать сетевой трафик на эту виртуальную машину.</span><span class="sxs-lookup"><span data-stu-id="eb1de-497">Otherwise, the load balancer won't route network traffic to that VM.</span></span>

<span data-ttu-id="eb1de-507">**Используйте управляемые диски для VHD.**</span><span class="sxs-lookup"><span data-stu-id="eb1de-507">**Use Managed Disks for VHDs.**</span></span> <span data-ttu-id="eb1de-508">Служба [Управляемые диски][managed-disks] обеспечивает более высокую надежность виртуальных машин в группах доступности, так как диски достаточно изолированы друг от друга, что позволяет избежать единой точки отказа.</span><span class="sxs-lookup"><span data-stu-id="eb1de-508">[Managed Disks][managed-disks] provide better reliability for VMs in an availability set, because the disks are sufficiently isolated from each other to avoid single points of failure.</span></span> <span data-ttu-id="eb1de-509">Кроме того, управляемые диски не подпадают под ограничения операций ввода-вывода в секунду для виртуальных жестких дисков (VHD), созданных в учетной записи хранения.</span><span class="sxs-lookup"><span data-stu-id="eb1de-509">Also, Managed Disks aren't subject to the IOPS limits of VHDs created in a storage account.</span></span> <span data-ttu-id="eb1de-510">Дополнительные сведения см. в статье [Управление доступностью виртуальных машин Windows в Azure][vm-manage-availability].</span><span class="sxs-lookup"><span data-stu-id="eb1de-510">For more information, see [Manage the availability of Windows virtual machines in Azure][vm-manage-availability].</span></span>

<span data-ttu-id="eb1de-516">**Включите журналы диагностики**, в том числе базовые метрики работоспособности, а также ведение журналов инфраструктуры и [диагностику загрузки][boot-diagnostics].</span><span class="sxs-lookup"><span data-stu-id="eb1de-516">**Enable diagnostic logs**, including basic health metrics, infrastructure logs, and [boot diagnostics][boot-diagnostics].</span></span> <span data-ttu-id="eb1de-517">Если виртуальную машину невозможно загрузить, для обнаружения неисправностей можно использовать диагностику загрузки.</span><span class="sxs-lookup"><span data-stu-id="eb1de-517">Boot diagnostics can help you diagnose a boot failure if your VM gets into a non-bootable state.</span></span> <span data-ttu-id="eb1de-518">Дополнительные сведения см. в статье [Сбор и использование данных журнала из ресурсов Azure][diagnostics-logs].</span><span class="sxs-lookup"><span data-stu-id="eb1de-518">For more information, see [Overview of Azure Diagnostic Logs][diagnostics-logs].</span></span>

<span data-ttu-id="eb1de-519">**Используйте расширение AzureLogCollector.**</span><span class="sxs-lookup"><span data-stu-id="eb1de-519">**Use the AzureLogCollector extension.**</span></span> <span data-ttu-id="eb1de-520">(Только для виртуальных машин Windows.) This extension aggregates Azure platform logs and uploads them to Azure storage, without the operator remotely logging into the VM.</span></span> <span data-ttu-id="eb1de-521">Дополнительные сведения см. в статье [Расширение AzureLogCollector](/azure/virtual-machines/virtual-machines-windows-log-collector-extension/?toc=%2fazure%2fvirtual-machines%2fwindows%2ftoc.json).</span><span class="sxs-lookup"><span data-stu-id="eb1de-521">For more information, see [AzureLogCollector Extension](/azure/virtual-machines/virtual-machines-windows-log-collector-extension/?toc=%2fazure%2fvirtual-machines%2fwindows%2ftoc.json).</span></span>
