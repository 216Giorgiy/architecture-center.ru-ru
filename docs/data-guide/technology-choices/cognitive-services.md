# <a name="choosing-a-microsoft-cognitive-services-technology"></a>Выбор технологии Microsoft Cognitive Services

Microsoft Cognitive Services — это облачные интерфейсы API, которые можно использовать в приложениях искусственного интеллекта (ИИ) и потоках данных. В этих службах доступны предварительно обученные модели, которые можно использовать в приложении. При этом с вашей стороны не требуются ни данные, ни обучение моделей. Службы Cognitive Services разработаны командой Майкрософт, которая занимается искусственным интеллектом и исследованиями. В этих службах реализованы новейшие алгоритмы глубокого обучения. Они используются через интерфейсы HTTP REST. Кроме того, для многих распространенных платформ разработки приложений доступны пакеты SDK.

Компоненты Cognitive Services:

- Анализ текста
- компьютерное зрение;
- видеоаналитика;
- распознавание и генерация речи;
- распознавание естественного языка;
- интеллектуальный поиск.

Основные преимущества:

- простая разработка современных служб ИИ;
- простая интеграция в приложения через интерфейсы HTTP REST;
- встроенная поддержка Cognitive Services в Azure Data Lake Analytics.

Рекомендации:

- Службы доступны только через Интернет. Обычно требуется подключение к Интернету. Исключение — Пользовательская служба визуального распознавания, обученную модель которой можно экспортировать для прогнозирования на устройства и в IoT Edge.

- Хотя поддерживается широкая настройка, доступные службы могут не соответствовать всем требованиям к прогнозной аналитике.

## <a name="what-are-your-options-when-choosing-amongst-the-cognitive-services"></a>Варианты при выборе Cognitive Services

<!-- markdownlint-disable MD026 -->

В Azure есть десятки вариантов Cognitive Services. Актуальный список этих служб доступен в каталоге и классифицирован по функциям, которые они поддерживают:

- [зрение](https://azure.microsoft.com/services/cognitive-services/directory/vision/);
- [речь](https://azure.microsoft.com/services/cognitive-services/directory/speech/);
- [знания](https://azure.microsoft.com/services/cognitive-services/directory/know/);
- [Поиск](https://azure.microsoft.com/services/cognitive-services/directory/search/)
- [Язык](https://azure.microsoft.com/services/cognitive-services/directory/lang/)

## <a name="key-selection-criteria"></a>Основные критерии выбора

Чтобы ограничить количество вариантов, сначала ответьте на следующие вопросы:

- С какими типами данных вы работаете? Ограничьте количество вариантов с учетом типа входных данных, с которыми вы работаете. Например, если используются текстовые входные данные, выберите службы, которые работают с текстом.

- У вас есть данные для обучения модели? Если да, рекомендуем настраиваемые службы, которые для повышения точности и производительности позволяют обучать свои базовые модели с помощью данных пользователей.
